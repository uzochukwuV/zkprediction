# UltraHonk Verifier Integration

This document describes the integration of the real Barretenberg FFI bindings for actual ZK proof verification in the Soroban contract.

## Overview

The zkPrediction contract uses UltraHonk proofs generated by Noir circuits. The verification is performed on-chain using Soroban's native BN254 host functions.

## Tools & Versions

| Tool | Version |
|------|---------|
| Noir | 1.0.0-beta.9 |
| Barretenberg (bb) | 0.87.0 |
| Soroban SDK | 26.1.0 |
| Rust | 1.75+ |

## Verifier Crate

Location: `yugocabrio-rs-soroban-ultrahonk/crates/ultrahonk-soroban-verifier/`

The verifier is based on the [ProofBridge rs-soroban-ultrahonk](https://github.com/Explore-Beyond-Innovations/rs-soroban-ultrahonk) crate.

### Key Features

- **No_std compatible** - Works in Soroban contract environment
- **Native BN254 operations** - Uses Soroban host functions:
  - `env.crypto().bn254().g1_msm()` - Multi-scalar multiplication
  - `env.crypto().bn254().pairing_check()` - Final pairing verification
- **Fiat-Shamir transcript** - Uses keccak hash for challenge generation

## Verification Pipeline

1. **Parse VK** - Load verification key from bytes (1760 bytes)
2. **Parse Proof** - Load proof from bytes (14592 bytes = 456 field elements)
3. **Validate Public Inputs** - Check length matches VK metadata (64 bytes = 2 inputs)
4. **Generate Transcript** - Fiat-Shamir with keccak
5. **Compute Public Input Delta** - Fold public inputs into relation parameters
6. **Sumcheck Verification** - Verify sumcheck polynomial relations
7. **Shplemini Opening** - G1 MSM for commitment folding
8. **Pairing Check** - Final BN254 pairing verification

## Proof Generation

```bash
# Compile circuit
nargo compile

# Generate witness
nargo execute

# Generate VK
bb write_vk -s ultra_honk --oracle_hash keccak -b target/prediction_settle.json -o target/claim/vk

# Generate proof
bb prove -s ultra_honk --oracle_hash keccak \
  --bytecode_path target/prediction_settle.json \
  --witness_path target/prediction_settle.gz \
  --output_path target/claim/proof.bin
```

## Integration with Contract

```rust
// contracts/prediction/src/lib.rs
use verification::UltraHonkVerifier;

fn verify_proof_bytes(env: &Env, proof: Bytes, public_inputs: Bytes) -> bool {
    let vk_bytes = env.storage().instance()
        .get::<_, Bytes>(&DataKey::VkBytes)
        .expect("VK bytes not set");

    let verifier = match UltraHonkVerifier::new(env, &vk_bytes) {
        Ok(v) => v,
        Err(_) => return false,
    };

    verifier.verify(&proof, &public_inputs).is_ok()
}
```

## Troubleshooting

### Proof Verification Fails

1. Ensure proof was generated with matching toolchain versions
2. Verify VK and proof were generated with same circuit
3. Check `--scheme ultra_honk --oracle_hash keccak` flags

### VK Parse Error

- Ensure VK is 1760 bytes (bb v0.87.0 format)
- Check circuit compilation succeeded
